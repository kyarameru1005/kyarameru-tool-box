# Project AGENTS Policy

このファイルは `kyarameru-tool-box` リポジトリ専用ルールを定義する。

## 目的と優先順位

- 目的: リポジトリ運用時の判断基準を固定し、実装とレビューのブレを減らす。
- 優先順位は `正確性 > 安全性 > 速度` とする。

## 基本

- 応答は原則日本語（英語指定時のみ英語）。
- 回答は簡潔に、結論と次アクションを先に示す。
- 不確かな情報は断定せず、必要時は調査して根拠を示す。

## ファイル構造

- ドキュメントは `docs/` 配下に置く。
- 実行スクリプトは `scripts/` 配下に置く。
- 配備対象資材は `toolbox/` 配下に置く。
- GitHub Actions は `.github/workflows/` に置く。

## 命名規則

- 新規ファイル名の単語区切りはハイフン（kebab-case）を使う。
- 新規ファイル名でアンダースコア（snake_case）は使わない。
- 既存ファイルは互換性のため維持してよい。

## Git / PR ルール

- `main` へ直接 push せず、PR 経由で反映する。
- 破壊的操作（`git reset --hard` など）は明示依頼時のみ実行する。
- PR本文に「目的 / 変更点 / 検証結果」を記載する。

## ログ運用

- 作業記録の正本はPR本文とする。
- `ai_log` は PR 化しない作業（調査メモ・一次対応）に限って使用する。
