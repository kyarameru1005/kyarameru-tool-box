# Global AGENTS Policy

この文書は、Codex運用時の実行ルールを定義する。
優先順位は `正確性 > 安全性 > 速度` とする。

## 1. 応答ルール

- 応答は原則日本語とし、英語指示がある場合のみ英語で返す。
- 回答は「結論 -> 根拠 -> 次アクション」の順で簡潔に示す。
- 不確かな情報は断定しない。必要時は調査して根拠を明示する。

## 2. 実行ルール

- 実装前に関連ファイルを確認し、推測ではなく現状コードを根拠に判断する。
- プロジェクト内ルールは `AGENTS.md` -> `README.md` -> `docs/` の順で参照する。
- 影響のある変更を行った場合は、実行した検証手順と結果を必ず報告する。
- 外部反映を伴う操作（push、PR作成、GitHub設定変更）は実施内容を明示して実行する。

## 3. Git / 安全性ルール

- `main` への直接pushは禁止し、変更はPR経由で反映する。
- 破壊的操作（`git reset --hard`、履歴改変、強制上書き）は明示依頼がある場合のみ実行する。
- 自分が加えていない差分を勝手に戻さない。
- コミットとPRには、少なくとも「目的」「主な変更点」「検証結果」を残す。

## 4. ログ運用ルール

- 作業記録の正本はPR本文とする。
- `ai_log` は、PR化しない作業（ローカル検証メモ、暫定調査、障害一次対応）に限って使用する。
- 通常のPR作業では `ai_log` を更新しない。

## 5. 適用手順

- このファイルを更新したら `python3 scripts/install.py update` で `~/.codex/AGENTS.md` に反映する。
- 反映時に既存 `~/.codex/AGENTS.md` は `AGENTS.md.bak.<timestamp>` としてバックアップされる。

## 6. ファイル命名 / 構造ルール

- 新規ファイル名の単語区切りはハイフン（kebab-case）を使用する（例: `harness-plan.md`）。
- アンダースコア（snake_case）は新規ファイル名に使用しない（既存ファイルは互換性のため維持可）。
