# agents-md-writer

目的: 各リポジトリ向け `AGENTS.md` を、ベストプラクティス中心のヒアリングで作成し、最後に自己チェックまで行う。

## 推奨トリガー

- 「このリポジトリ用の AGENTS.md を作りたい」とき
- 「運用ルールを明文化したい」とき
- 「既存 AGENTS.md が曖昧でブレる」とき

## ワークフロー

1. リポジトリを確認する（`README.md` / `docs/` / 既存 `AGENTS.md`）。
2. ベストプラクティス観点でヒアリングする。
3. `AGENTS.md` 草案を生成する。
4. `scripts/check_agents_md.sh` で自己チェックする。
5. チェック結果を反映して最終版を提示する。

## ヒアリング項目（必須）

- このリポジトリの目的と優先順位（正確性/安全性/速度）
- 言語方針（日本語/英語）
- Git運用（main 直push可否、PR必須条件）
- 命名規則（kebab-case 等）
- ログ方針（PR本文正本か、補助ログの扱い）
- 禁止操作と要確認操作（破壊的操作、外部反映）

## 出力要件

- `AGENTS.md` は次の内容を含める:
  - 目的/優先順位
  - 応答ルール
  - 実行ルール
  - Git/安全性ルール
  - ログ運用
  - ファイル命名ルール
- ルールは判定可能な文で書く（曖昧語を避ける）。
