# AGENTS.md Best Practices

## 目的

`AGENTS.md` は「エージェントが迷わず実行判断できるルール」を提供する。

## 設計原則

- 優先順位を固定する（例: 正確性 > 安全性 > 速度）。
- 抽象語を避け、条件と行動を明示する。
- グローバルルールとリポジトリ固有ルールを分離する。
- 出力の型を固定する（例: 目的/変更点/検証結果）。

## 推奨セクション

1. 目的と優先順位
2. 応答ルール
3. 実行ルール
4. Git / 安全性
5. ログ運用
6. 命名規則（必要なら構造ルール）

## 曖昧語の例（避ける）

- 適宜
- 必要に応じて
- 可能であれば
- 状況に応じて

## 良い記述例

- 悪い例: 「適宜テストする」
- 良い例: 「変更を加えたら `python3 -m pytest -q` を実行し、結果を報告する」

- 悪い例: 「必要に応じて確認する」
- 良い例: 「push / PR作成 / GitHub設定変更の前に実施内容をユーザーへ明示する」

## 完了条件

- 必須セクションがすべて存在する。
- 曖昧語が含まれない。
- 命名規則とログ方針が明文化されている。
